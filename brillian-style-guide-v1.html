Design system · v1.0

Brillian

Tokens, components, and usage rules for the Brillian business valuation and financial health platform. Built as a two-layer Chakra system — primitive color ramps underneath, semantic role-based tokens on top. Components consume the semantic layer only.

TypefaceGoogle Sans

02
Semantic tokensSemantic

The interface components consume. Every semantic token is a reference to a primitive — components never read raw blue.500, they read brand.primary. This is what lets a rebrand, dark mode, or print variant happen without touching component code.

Architecture rule Components reference semantic tokens. Semantic tokens reference primitives. Primitives hold the hex values. Don't skip a layer.

Border radius

Six radii cover every surface in the system. Buttons get a sharp radius.button (4px) that sets them apart from pills — a deliberate corner language that reads more financial-product, less consumer-app. Cards default to radius.lg, pills always use radius.pill, micro-elements use radius.sm.

12
Usage rules

The system holds together because saturated colors are rare. These rules govern where each color tier can appear.

Status / secondary palette
✓ Where it can appear
  • Status pills and tags (≤11px text)
  • Status dots (≤9px diameter)
  • Icon tints (tinted bg + dark glyph)
  • Micro-bars and sparkline strokes
  • Single delta numbers (↑ +12%)
  • Categorical chart segments (≤4 at once)
✕ Where it never appears
  • Hero cards or full-bleed surfaces
  • Card backgrounds or wrappers
  • Page or section backgrounds
  • Buttons, CTAs, primary actions
  • Headlines or body text
  • Borders or dividers
Color hierarchy
1 Primary — brand blue
  • Hero valuation cards
  • Primary CTAs and buttons
  • Health score rings
  • Eyebrows and section markers
  • Benchmark bar fills
2 Structural — navy
  • Report header bands
  • Score-card backgrounds
  • Headlines and primary text
  • Deep accent surfaces
  • Print-context anchors
3 Growth — lime
  • Excess cash flow figures
  • Positive YoY deltas
  • "Growing" / "Above target" badges
  • Forward-momentum signals
4 Status — secondary
  • Five-state callouts only
  • Always small surfaces
  • Always paired tint + dark for legibility
  • Never a substitute for primary blue
